# Example Project: Object Typeface

## Overview

The first example project is a typeface created from an everyday object.

The user sees possible letterforms in the object when it is placed in different positions. The object has a small number of connected parts, a limited range of movement, and a consistent material quality. Those constraints become the starting point for a type system.

The project starts with observation:

- photos of the object
- notes about what the user sees
- possible letters or numbers suggested by different positions
- descriptions of the object's movement limits
- descriptions of the object's material, shape, and texture

Codex helps turn those observations into a structured typeface project.

## Why This Example Matters

This project explores how AI tools can help people move between creative observation and working software.

An artist may see a shape, texture, movement, or pattern in an object. An engineer may see rules, constraints, artifacts, and systems. This project uses the app to hold both views together.

The app should help a user start with something they noticed in the real world and develop it toward a usable digital product.

The studio is also a place to test whether an idea should continue. The user can shelve the project at any time and start a new one.

## Creative Direction

The source object has a manufactured, metallic quality. It suggests a grey graphite or pencil-like visual mood, while still feeling more engineered than hand-drawn.

Because the same object is used across the letters, the resulting typeface should have a shared visual language. Some glyphs may map naturally to the object. Others may need interpretation.

The first useful direction may be an uppercase display typeface. This can be tested before expanding into lowercase, punctuation, symbols, arrows, weights, italics, or other variants.

## Project Inputs

Initial inputs can include:

- object photos from different angles
- short videos showing how the object moves
- spoken notes or transcripts
- written observations
- early guesses about letters, numbers, or symbols
- notes about where the object does not naturally fit a glyph

The user does not need to define every glyph manually. The app should help identify the object's rules and use those rules to suggest the missing parts of the system.

## Codex Guidance

After the user adds initial material, Codex should produce a first guidance pass.

For this project, that guidance can include:

- what visual rules Codex sees in the object
- which letters or numbers may be natural starting points
- which glyphs are likely to be difficult
- what source material would help next
- what a first glyph set should include
- how to test whether the typeface has a coherent visual system

The first response should stay light. It should give the user a high-level path, such as three broad stages, without trying to teach the whole discipline at once.

Learning should happen just in time. As the user works inside each stage, the app can explain the relevant type design concerns, ask for missing context, and show why a next action matters.

The user should feel like they are collaborating with the product. The app should help the user learn enough to make the next useful decision.

## First Output

The first output does not need to be a finished font.

A useful first output is:

- a typeface concept summary
- a set of design rules
- a first glyph plan
- a list of source material still needed
- a small next action

For the typeface example, a usable test artifact is high priority. The user should be able to get the font in front of someone quickly, even if the glyph set is limited.

Useful test outputs can include:

- SVG glyph drafts
- a web font preview
- an installable font file
- specimen pages
- font build files
- web font files
- showcase previews for feedback

The purpose is fast review. A reviewer should be able to try the typeface in a real setting, such as a web page or a document, while the idea is still fresh enough to discuss.

## Studio and Showcase Surfaces

The studio contains the raw project material:

- original photos
- protected notes
- design reasoning
- generated drafts
- Codex run history

The showcase can show selected outputs:

- a specimen image
- a preview word
- a single glyph
- a partial alphabet
- a web type sample
- a released font package

This supports a product studio workflow where early creative IP stays protected until the user chooses to preview, review, or release it.

## Demo Story

The demo can show a user starting with a simple observation:

> I see possible letters in this object when I move it into different positions.

The user adds notes and object references. The app starts the first Codex guidance pass. Codex turns the loose idea into a structured path for creating a typeface.

The project then has a saved direction, a next action, and a protected record of how the idea is developing.
